## Overview
Merge all feature branches (`feat/udp-transport-hwdec`, `feature/usb-webcam`, `feature/rpicam-support`, `feature/gamepad-input`, `feature/monitor-mode`) into a unified codebase on `master`, combining functionality while filtering out debug/test code.

## Architecture

### Core Protocol Layer (rpv-proto)
- L2 header (8 bytes): Magic + Drone ID + Payload Type + Sequence
- Payload types: VIDEO (0x01), TELEMETRY (0x02), RC (0x03), HEARTBEAT (0x04), MAVLINK (0x05)
- MAX_PAYLOAD: 1400 bytes
- Reed-Solomon 4+2 FEC for video
- Raw socket common utilities (radiotap parsing, RSSI)

### Transport Modes (Pluggable)
1. **Raw 802.11 Monitor Mode** (default for production)
   - Direct WiFi frame injection/reception
   - Requires monitor-mode capable adapter
   - Lowest latency
   
2. **UDP/IP** (development/testing)
   - Works over standard networks
   - Discovery protocol (port 9002)
   - Data port 9001
   - Easier to debug

### Video Sources (Selectable)
1. **CSI Camera** (rpicam-vid) - Raspberry Pi
2. **USB Webcam** (ffmpeg) - Any UVC device
3. **Generic Video** (ffmpeg) - File, stream, or device

### Input Methods (Optional)
1. **MAVLink over UART** - Flight controller telemetry/RC
2. **Gamepad/Joystick** (evdev) - Direct RC input
3. **File-based RC** - Fallback (/tmp/rpv_rc_channels)

## Build Configuration

### Cargo Features
```toml
[features]
default = ["raw-sock", "udp-transport"]
raw-sock = []          # Raw 802.11 monitor mode
udp-transport = []     # UDP/IP transport
usb-webcam = []        # USB webcam support
rpicam = []            # Raspberry Pi Camera support
gamepad = []           # Gamepad/Joystick input
systemd = []           # Systemd service files
```

### Conditional Compilation
- Use `#[cfg(feature = "...")]` for feature-specific code
- Common protocol in `rpv-proto` (always compiled)
- Transport selection in main.rs
- Video source selection in video_tx.rs

## Implementation Steps

### Phase 1: Protocol Layer (rpv-proto)
- [x] Already consolidated in feat/udp-transport-hwdec
- Contains: link.rs, rawsock_common.rs, udpsock.rs, config.rs, discovery.rs
- No changes needed

### Phase 2: Cargo.toml Unification
Create workspace Cargo.toml with:
- Feature flags for each optional component
- Dependencies conditional on features
- Common dependencies (tokio, crossbeam, etc.) always included

### Phase 3: rpv-cam Consolidation
Merge from multiple sources:
1. **Main entry point** (from udp-transport-hwdec)
   - Transport selection (raw vs UDP)
   - Discovery protocol integration
   - Thread spawning (RX, video, telemetry, heartbeat)
   
2. **Video capture** (from rpicam-support + usb-webcam)
   - rpicam-vid integration (CSI camera)
   - ffmpeg capture (USB/webcam)
   - Configurable via camera_type parameter
   
3. **FC interface** (from all branches)
   - MAVLink over UART
   - Serial port configuration
   - Telemetry parsing
   
4. **Raw socket** (from monitor-mode)
   - AF_PACKET implementation
   - Radiotap header construction
   - TX/RX with RSSI

5. **Remove**: Debug logging, duplicate code, test configurations

### Phase 4: rpv-ground Consolidation
Merge from multiple sources:
1. **Main entry point** (from udp-transport-hwdec)
   - Transport selection
   - Discovery protocol
   - UI setup (egui + wgpu)
   
2. **Video pipeline** (from udp-transport-hwdec)
   - FEC reassembly (receiver.rs)
   - H.264 decoder (decoder.rs)
   - GPU YUV→RGB (YuvGpuResources)
   
3. **Gamepad input** (from gamepad-input)
   - evdev integration
   - Hot-plug detection
   - Channel mapping
   
4. **Link state machine** (from udp-transport-hwdec)
   - Heartbeat monitoring
   - Signal lost/connected states
   
5. **Telemetry** (from all branches)
   - JSON parsing
   - MAVLink bridge (QGC)
   
6. **Remove**: Debug logging, duplicate functions, test code

### Phase 5: Configuration
- Unified config format (TOML)
- Common section (interface, drone_id, transport, etc.)
- Camera-specific section (framerate, bitrate, etc.)
- Ground-specific section (GCS ports, etc.)

### Phase 6: Deployment Scripts
- Update install-cam.sh for unified binary
- Update install-ground.sh for unified binary
- Systemd services support all features
- Network setup scripts handle both raw and UDP modes

### Phase 7: Testing
- Build in release mode
- Test raw socket mode (requires monitor-capable WiFi)
- Test UDP mode (works on any network)
- Verify video streaming
- Verify telemetry/RC
- Verify OSD rendering

## Debug Code to Remove

### High-Volume Logging
- Frame-by-frame debug logs (decoder.rs, receiver.rs)
- Hex dumps of raw frames
- Excessive tracing::info! in hot paths
- Keep: Error logs, state changes, periodic stats

### Test Configurations
- Hardcoded test resolutions
- Test bitrates/framerates
- Development-only network configs

### Duplicate Code
- Multiple implementations of same function
- Unused utility functions
- Commented-out code blocks

### Incomplete Features
- Partially implemented codecs
- Experimental features without tests
- TODO/FIXME comments without associated issues

1. **Build Failures**: Use Cargo features for conditional compilation
2. **Runtime Errors**: Extensive error handling, graceful degradation
3. **Performance**: Profile release builds, optimize hot paths
4. **Compatibility**: Maintain backward compatibility where possible
5. **Testing**: Test each feature combination separately
